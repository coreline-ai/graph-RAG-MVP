# Hybrid GraphRAG PRD

기준 데이터: `data/chat_logs*.txt` 분석 반영

- 전체 로그: 10,000건
- 샘플 파일: 100건
- 날짜 범위: 2024-01-01 ~ 2026-03-23
- 채팅방 수: 30개
- 사용자 수: 179명
- 메시지 본문 평균 길이: 약 17자
- 메시지 본문 최대 길이: 26자
- 본문 내부 `, ` 포함 메시지: 569건
- malformed line: 0건

## 1. 문서 개요

### 1.1 목적

본 문서는 한국어 사내 채팅 로그를 대상으로 하는 Hybrid GraphRAG 시스템의 제품 요구사항을 정의한다. 본 시스템은 날짜, 채팅방, 사용자 같은 구조화 조건 검색과 의미 기반 검색을 결합하고, 검색 결과 주변 맥락과 집계 인사이트를 함께 제공하는 내부 탐색 도구를 목표로 한다.

### 1.2 제품 한 줄 정의

짧은 한국어 운영성 채팅 로그를 대상으로, 메타데이터 필터 + 의미 검색 + 그래프 기반 맥락 확장을 제공하는 내부 검색 및 인사이트 시스템.

### 1.3 배경

현재 기준 데이터는 이미 정규화된 TXT 로그 포맷으로 저장되어 있으나, 다음과 같은 한계가 있다.

- 특정 날짜와 특정 채팅방 조건을 동시에 만족하는 대화를 빠르게 찾기 어렵다.
- 키워드를 정확히 모를 때 의미 기반 탐색이 어렵다.
- 개별 메시지만 찾으면 앞뒤 문맥과 같은 방/같은 사용자 관점의 맥락을 파악하기 어렵다.
- 운영 패턴, 배포/롤백/장애/PR 같은 활동 흐름을 기간 단위로 보기 어렵다.

### 1.4 제품 전략

본 제품은 `운영 검색층`과 `인사이트 확장층`을 가진 Hybrid GraphRAG 방향을 따른다.

- MVP 구현 중심은 `Message`, `User`, `Room`, `Date` 중심의 Lightweight GraphRAG다.
- 검색 품질은 메타데이터 필터, 의미 검색, 그래프 맥락 확장으로 확보한다.
- Microsoft GraphRAG 스타일의 대규모 요약 그래프는 현재 데이터 특성과 범위를 고려해 후속 확장으로 남긴다.

## 2. 문제 정의

한국어 사내 채팅 로그는 짧고 단문 위주이며, 날짜와 채팅방이라는 강한 메타데이터를 가진다. 그러나 기존 단순 문자열 검색만으로는 다음 문제를 해결하지 못한다.

- 날짜 조건과 의미 검색을 동시에 만족하는 결과를 안정적으로 반환하기 어렵다.
- 같은 방에서 연속된 운영 이슈 흐름을 따라가기가 어렵다.
- 특정 사용자 또는 특정 주제의 활동 패턴을 구조적으로 집계하기 어렵다.
- 검색 결과가 단일 메시지에 머물러 실제 의사결정 근거로 쓰기 어렵다.

## 3. 목표와 비목표

### 3.1 제품 목표

본 MVP는 다음 두 목표를 동등 우선순위로 가진다.

1. 사내 검색 MVP
   - 날짜, 채팅방, 사용자 필터와 의미 검색을 결합해 근거 메시지를 빠르게 찾는다.
2. 인사이트 탐색 MVP
   - 기간별/방별/사용자별 활동 패턴과 대표 메시지를 구조화해 보여준다.

### 3.2 세부 목표

- 메타데이터 필터 검색을 정확하게 수행한다.
- 짧은 한국어 문장에 대해 의미 검색을 제공한다.
- 검색 결과에 인접 맥락을 함께 노출한다.
- 집계 카드와 대표 메시지를 통해 운영 흐름을 탐색할 수 있게 한다.

### 3.3 비목표

다음 항목은 MVP 범위에서 제외한다.

- Slack, KakaoTalk, Teams 등 원본 메신저 export 파싱
- 문서/티켓/이메일 등 멀티소스 수집
- 자동 엔터티 추출 기반 대규모 Knowledge Graph 구축
- LLM 기반 자유서술 요약 및 서술형 리포트 생성
- 권한 ACL 완전 구현
- 외부 사용자용 서비스 제공

## 4. 대상 사용자

### 4.1 핵심 사용자군

- 운영팀: 장애, 롤백, 배포, 모니터링 관련 과거 대화 추적
- 개발팀: PR, 이슈, 마이그레이션, API 성능 관련 히스토리 확인
- 분석/기획팀: 특정 기간의 활동 패턴과 방별 주제 흐름 파악
- 관리자/리더: 팀/방 단위의 빈도와 대표 사례를 빠르게 점검

### 4.2 사용자 니즈

- 정확한 날짜 범위를 걸고 원하는 메시지를 찾고 싶다.
- 키워드가 불완전해도 비슷한 의미의 메시지를 찾고 싶다.
- 결과 메시지 전후의 운영 맥락을 바로 보고 싶다.
- 특정 기간에 어떤 방과 사용자가 활발했는지 보고 싶다.

## 5. 기준 데이터와 제품 함의

### 5.1 데이터 특성

현재 입력 데이터는 아래 포맷의 1라인 1메시지 텍스트다.

`[날짜, 시간, 채팅방, 내용, 사용자]`

실제 데이터 분석 결과는 다음과 같다.

- 메시지는 대부분 짧은 운영성 문장이다.
- `배포`, `롤백`, `장애`, `PR #`, `이슈 #`, `API` 같은 운영 키워드가 반복적으로 등장한다.
- 메시지 내부 쉼표가 569건 존재하므로 단순 CSV 파싱으로는 정확성을 보장할 수 없다.
- malformed line은 현재 없지만, 파서 실패 전략은 반드시 필요하다.

### 5.2 제품 설계에 주는 영향

- 검색 기본 단위는 `메시지 1건`으로 둔다.
- 긴 문서 청킹보다 메타데이터와 짧은 문장 임베딩 품질이 중요하다.
- 검색 결과에는 같은 방 기준 인접 메시지를 붙여 문맥을 보완해야 한다.
- 인사이트는 자유서술 요약보다 집계와 대표 메시지 노출이 우선이다.

## 6. 핵심 기능

### 6.1 메타데이터 필터 검색

- 날짜 범위 필터
- 채팅방 다중 선택 필터
- 사용자 다중 선택 필터
- 필터 조건 완전 일치 보장

### 6.2 의미 검색

- 키워드가 정확하지 않아도 유사 의미 메시지 검색
- 짧은 한국어 운영성 문장에 최적화된 검색
- full-text 검색과 벡터 검색의 결합 결과 제공

### 6.3 그래프 컨텍스트 확장

- 같은 방의 이전/다음 메시지
- 같은 사용자의 최근 메시지
- 같은 날짜, 같은 방의 관련 메시지 묶음

### 6.4 인사이트 집계 보기

- 기간별 메시지 수
- 방별 활동량
- 사용자별 활동량
- 특정 운영 키워드가 포함된 대표 메시지 샘플

## 7. 사용자 시나리오

### 7.1 특정 날짜 배포 검색

사용자는 `2024-01-05` 전후 배포 관련 메시지를 찾고 싶다. 시스템은 날짜 범위 필터와 의미 검색을 함께 사용해 `"서버 배포 380차 완료했습니다"` 같은 메시지를 반환하고, 같은 방의 전후 맥락도 함께 보여준다.

### 7.2 특정 방 장애 히스토리 확인

사용자는 `신규사업TF` 방에서 장애 대응 메시지 흐름을 보고 싶다. 시스템은 방 필터와 검색어 `"장애 대응"`을 사용해 관련 메시지를 찾고, 같은 방의 인접 메시지를 보여준다.

### 7.3 특정 사용자 활동 확인

사용자는 특정 사용자의 최근 활동 패턴과 어떤 방에서 주로 발화했는지 보고 싶다. 시스템은 사용자 필터와 인사이트 화면을 통해 활동량과 대표 메시지를 제공한다.

### 7.4 롤백/배포/PR 패턴 탐색

사용자는 여러 방에서 `롤백`, `배포`, `PR #` 관련 메시지 빈도와 대표 사례를 보고 싶다. 시스템은 집계 카드와 대표 메시지 목록을 제공한다.

### 7.5 기간별 방 활성도 비교

사용자는 특정 기간 동안 어떤 채팅방이 가장 활발했는지 비교하고 싶다. 시스템은 방별 메시지 수와 대표 키워드 샘플을 보여준다.

### 7.6 검색 결과 근거 메시지 열람

사용자는 검색 결과가 왜 나왔는지 확인하고 싶다. 시스템은 검색 결과 상세에서 원문 메시지, 메타데이터, 인접 메시지를 함께 보여준다.

## 8. 사용자 흐름

### 8.1 검색 화면

1. 사용자가 검색어를 입력한다.
2. 날짜 범위, 채팅방, 사용자 필터를 선택한다.
3. 시스템이 필터 검색 + 의미 검색을 수행한다.
4. 결과 목록에서 메시지, 점수, 방, 사용자, 시각을 보여준다.
5. 상세 보기에서 인접 메시지와 같은 사용자의 최근 메시지를 확인한다.

### 8.2 인사이트 화면

1. 사용자가 기간 필터를 설정한다.
2. 시스템이 기간별 메시지 수, 방별 활동량, 사용자 활동량을 집계한다.
3. 카드와 표에서 상위 항목을 보여준다.
4. 대표 메시지 샘플을 통해 실제 근거를 확인한다.

## 9. 비기능 요구사항

- 검색 응답시간: 10,000 메시지 기준 로컬 개발 환경에서 p95 2초 이하
- 인사이트 집계 응답시간: 10,000 메시지 기준 p95 3초 이하
- 파서 정확도: 정상 라인 100% 파싱
- 근거 메시지: 검색 결과는 항상 원문 메시지를 포함
- 필터 정확도: 날짜/방/사용자 필터 오탐 0
- 시스템 가시성: 적재 건수, 실패 건수, 인덱스 상태 확인 가능

## 10. 성공 지표

- Top-5 recall
- 필터 오탐 0
- 검색 응답시간 p95 2초 이하
- 인사이트 화면에서 사용자가 근거 메시지까지 2클릭 이내 도달
- 적재 성공률 100% (정상 데이터 기준)
